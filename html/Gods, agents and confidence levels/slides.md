# Perkūnas: AI vs. Mythology

Looking good is not the same as being correct

- **Perkūnas**: Baltic god of thunder, sky, storms, rain, and war.
- **AI-Generated Image**: Shows him riding a carriage pulled by three goats, holding an axe.
- **The Historical Reality**:
  - Goats were *sacrificed* to Perkūnas; he doesn't ride them.
  - He traditionally wields a *sword*, not an axe.
- **Key Lesson**: Visually appealing outputs from AI can be factually wrong.

# Shifting from Agents to Skills

Decoupling identity from action sequences

- **Agents**: Define *who* they are, their persona, and what tools they can access.
- **Skills**: Define *how* a specific task is orchestrated.
- Split the monolithic logic into two reusable **Skills**:
  1. **Fact Check Skill** — verifying existing content.
  2. **Research Skill** — discovering and verifying new concepts.

<!-- notes:
- Explain the architectural shift: moving from heavy agents to modular skills.
- Like a person who has skills (cooking, typing) that are separate from their personality.
- This lets us reuse skills across different agents or even different projects.

# The Multitool Headache

The pain of configuration divergence

- Using multiple AI tools for different purposes:
  - **GitHub Copilot** (cheap prototyping and completions).
  - **Gemini & Antigravity** (quick fixes and research).
- **Divergence Nightmare**:
  - 5 agents + 2 skills = 7 configs.
  - Duplicated across `.github`, `.gemini`, and `.antigravity` folders.
  - **15 files to maintain manually**! Updates became inconsistent.

# The Solution: Symlinks

Single source of truth across all tools

- Move all agent and skill definitions to a central `.ai/` folder.
- Create **symbolic links** in each tool's configuration directory:
  ```bash
  .ai/agents -> .gemini/agents
  .ai/skills -> .gemini/skills
  # Same symlinks for .antigravity and .github
  ```
- **Unified Frontmatter**: Mixed frontmatter fields are gracefully parsed; tools ignore keys they don't recognize.

<!-- notes:
- Walk through the symlink setup.
- This simple trick of directory symlinking solved the duplication.
- Any change made in `.ai/` is instantly visible to all AI tools.
- Frontmatter differences are ignored by tools that don't support them.
-->

# Tool Quirks & Frontmatter

Handling differences between LLM clients

- **Gemini**:
  - Strict validator; crashes on unknown frontmatter fields.
  - *Fix*: Keep configs lean and rely on sensible defaults.
- **Claude**:
  - Powerhouse; supports 20+ frontmatter fields for advanced steering.
- **Strategy**: Design for the lowest common denominator, or use a pre-process script if needed.
